# J-TACS Sales Package

A premium, commercial-grade sales kit. Two artefacts, designed to be sent before — or shown during — a partner-level demo call.

| File | Use it when… |
|------|--------------|
| **`J-TACS_Sales_Deck.pdf`** | Sending ahead of a meeting; leaving behind after; printed for a partner who still likes paper. 15 pages, A4. |
| **`index.html`** | Driving a live demo on a 4K display; sharing a link; in a follow-up email. Single file, no build, no server — opens directly. |

Both use the same brand language, the same numbers, and the same closing call-to-action — so a prospect who looks at one and then the other doesn't feel like they're talking to two companies.

---

## How to send these

### As a cold inbound
Send `J-TACS_Sales_Deck.pdf` only. Subject line: *"How a 15-person CA firm gets back ₹45 lakh a year."* Body: one sentence + the deck.

### As a follow-up to a discovery call
Send both. Reference one number from page 3 (Cost of Doing Nothing) that matched their actual situation. Link the HTML for them to forward to their co-partner.

### On-screen during a demo
Open `index.html` full-screen. Start from the hero, scroll-tell-scroll. Park on Section 7 (ROI) and ask: *"Which of these three profiles is closest to you?"*

## Rebuilding the PDF

```bash
pip install reportlab
python scripts/build-sales-deck.py
# → sales/J-TACS_Sales_Deck.pdf
```

The script is deterministic. Edit copy in `scripts/build-sales-deck.py` and re-run.

---

## Rebuilding the HTML

It's a single file — `sales/index.html`. Open it in any modern browser. No build step. Embedded SVG logo, embedded CSS, embedded JS. To deploy, drop it on any static host (Vercel, Netlify, S3, or `python -m http.server` from `/sales`).

To customise pricing, search for `PROFESSIONAL` in the HTML and the four `.price-card` blocks follow each other. ROI numbers live in three `.roi-card` blocks just above.
